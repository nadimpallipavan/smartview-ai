# Deploying SmartView AI to Render

This repo includes a [`render.yaml`](render.yaml) Blueprint that provisions everything:
**web (Puma)** + **worker (Sidekiq)** + **PostgreSQL** + **Redis** ‚ÄĒ all on Render's free tier.

## One-time deploy (‚Čą5 minutes)

1. **Sign in to Render** ‚Üí https://dashboard.render.com (sign up with GitHub ‚ÄĒ free).

2. **New Blueprint**
   - Click **New +** ‚Üí **Blueprint**
   - Connect your GitHub account if prompted, then select the **`smartview-ai`** repo.
   - Render reads `render.yaml` and shows the 4 resources it will create.
   - Click **Apply**.

3. **Wait for the build** (~5‚Äď8 min the first time)
   - Render builds the Docker image (installs gems, precompiles assets).
   - The web service starts; the `bin/docker-entrypoint` runs `db:prepare` (creates tables + runs migrations) automatically on first boot.

4. **Seed demo data** (one time)
   - Open the **smartview-web** service ‚Üí **Shell** tab ‚Üí run:
     ```bash
     bundle exec rails db:seed
     ```
   - This creates the demo accounts, 12 titles, and 8 live channels.

5. **Open your live app** at the URL Render assigns, e.g.
   `https://smartview-web.onrender.com`

   **Login:**
   - Admin: `admin@smartview.ai` / `password123`
   - Demo:  `demo@smartview.ai`  / `password123`

   ‚ö†ÔłŹ Change these passwords (or delete the seed accounts) before sharing publicly.

## Notes & options

- **Auto-generated secrets:** `SECRET_KEY_BASE` and the three Active Record encryption
  keys are generated by Render (shared across web + worker via the `smartview-secrets`
  env group). You don't set these manually.

- **Face recognition** runs in offline **mock mode** by default (`FACE_RECOGNITION_MOCK=true`).
  To use real AWS Rekognition, add these env vars to **both** the web and worker services
  in the Render dashboard and set `FACE_RECOGNITION_MOCK=false`:
  ```
  AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_REGION, AWS_REKOGNITION_COLLECTION_ID
  ```

- **Image uploads (Active Storage):** defaults to local disk (ephemeral on Render ‚ÄĒ files
  are lost on redeploy). For persistent media, set `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`,
  and `AWS_S3_BUCKET`; production then automatically uses S3. (Demo posters use placeholder
  image URLs, so this isn't required to see the app working.)

- **Free tier sleep:** free web services spin down after ~15 min of inactivity and take
  ~30‚Äď60s to wake on the next request. Free PostgreSQL expires after 30 days ‚ÄĒ upgrade to a
  paid instance for anything long-lived.

- **If Render rejects `type: redis`** in the blueprint, change it to `type: keyvalue`
  (Render renamed Redis to "Key Value"); everything else stays the same.

- **Future updates:** just `git push` to `main` ‚ÄĒ Render auto-deploys.

## Redeploying after code changes

```bash
# from inside the running web container (git lives there since git isn't on the host):
docker-compose exec web bash -c "cd /rails && git add -A && git commit -m 'your message' && git push"
```
Render detects the push and redeploys automatically.
